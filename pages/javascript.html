What is JavaScript?

JavaScript is a programming language that makes websites interactive. While HTML gives you the structure and CSS handles the styling, JavaScript lets you respond to user actions like clicks, typing, or form submissions. It runs directly in your browser without needing to reload the page.

Almost every modern website uses JavaScript for things like form validation, dropdown menus, image sliders, popups, and loading new content without refreshing. It is one of the three core technologies of the web, right alongside HTML and CSS.

Key Features of JavaScript

  • Client-side scripting (runs in the browser)
  • Event-driven programming
  • DOM manipulation (changing HTML/CSS on the fly)
  • Form validation before submitting
  • Asynchronous operations (fetching data)
  • Large ecosystem of libraries and frameworks

How JavaScript Works

When you load a webpage, the browser reads the HTML and builds a structure called the DOM (Document Object Model). JavaScript can then access that DOM and modify it. For example, when you click a button, JavaScript can show a hidden message, change colors, validate an email field, or fetch data from a server — all without reloading the page.

Code Snippet (JavaScript Validation)

function validateAndShow() {
    let name = document.getElementById('demoName').value;
    let email = document.getElementById('demoEmail').value;
    let password = document.getElementById('demoPassword').value;
    let confirmPwd = document.getElementById('demoConfirmPwd').value;
    let resultDiv = document.getElementById('demoResult');
    let errors = [];

    if(name === "" || name === null) {
        errors.push("Name is required");
    } else if(name.trim().length < 2) {
        errors.push("Name must be at least 2 characters");
    }

    if(email === "" || email === null) {
        errors.push("Email is required");
    } else if(!email.includes("@") || !email.includes(".")) {
        errors.push("Enter a valid email address");
    }

    if(password === "") {
        errors.push("Password is required");
    } else if(password.length < 6) {
        errors.push("Password must be at least 6 characters");
    }

    if(password !== confirmPwd) {
        errors.push("Passwords do not match");
    }

    if(errors.length > 0) {
        resultDiv.innerHTML = "ERROR: " + errors.join(" · ");
        resultDiv.style.background = "#fee";
        resultDiv.style.borderLeftColor = "#dc3545";
    } else {
        resultDiv.innerHTML = "Registration successful! Welcome " + name.trim();
        resultDiv.style.background = "#e6f7e6";
        resultDiv.style.borderLeftColor = "#28a745";
    }
}

Explanation of the Code

The JavaScript code above shows a real-world validation function. It:

  • Gets values from the input fields using document.getElementById()
  • Checks if the name is empty or too short (less than 2 characters)
  • Validates email format by checking for "@" and "." characters
  • Ensures password is at least 6 characters long
  • Compares password and confirm password fields
  • Collects all error messages into an array and displays them together
  • Shows a success message with the user's name if everything passes

This is typical of how client-side validation works on thousands of real websites — catching mistakes early before data is sent to a server.
